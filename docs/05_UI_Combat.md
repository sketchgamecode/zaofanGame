# 战斗交互与动效呈现标准设计案 (v1.1 官方实机校验版)
**适用场景:** 攻打州府 (PVE)、演武场 (PVP)
**设计宗旨:** 完美复刻 Shakes & Fidget 的极简战斗爽感 (Juice)，建立可全局复用的“剧本回放引擎”。

---

## 1. 核心架构：剧本回放模式 (Playback Engine)
战斗结果在逻辑层瞬间计算完成。前端 UI 接收 `TurnEvent[]` 剧本数组，通过定时器逐帧播放。
* **播放节奏:** 建议每回合动作间隔为 `600ms - 800ms`。
* **跳过机制 (Skip):** 屏幕底部中央常驻一个醒目的【跳过动画】按钮。点击后立即清除播放定时器，将血条瞬间扣至最终结果，并直接展示结算模态框。

---

## 2. 视觉布局 (Visual Layout)
采用经典的**绝对对称双阵营布局**，重点突出血量压迫感。

* **左侧区域 (Player) & 右侧区域 (Enemy):**
  * **顶部:** 角色大头像/立绘，以及名字与等级。
  * **中部 (核心视觉中心):** 极宽、极粗的生命值条 (HP Bar)，带有 CSS 宽度平滑过渡效果，并在血条中心用白色粗体显示当前具体血量数值。
  * **底部:** 静态的 5 大基础属性面板对齐展示，供玩家在战斗中直观比对双方硬件差距。

---

## 3. 动画与特效流 (Animation Pipeline)
放弃复杂的物理投射计算，采用 S&F 官方的“闪现劈砍”高性价比方案。

### 3.1 武器闪击 (Weapon Slash)
* 当判定一方发起攻击时，在**受击方**头像的正中央，瞬间生成一个攻击方武器的图标（剑、法杖或弓箭）。
* 该武器图标执行一个极短的 CSS 动画：迅速放大并伴随轻微旋转（模拟劈砍/射击），持续 `150ms` 后立刻销毁。
* 伴随武器消失，受击方头像可加上短暂的 CSS 滤镜闪红 (`filter: brightness(0.5) sepia(1) hue-rotate(-50deg) saturate(5);`) 或极小幅度的左右震动。

### 3.2 浮动战斗文字 (Floating Combat Text - FCT)
必须使用绝对定位覆盖在受击方头像区域，是最重要的爽感反馈。
* **通用样式:** 必须使用**粗体、深红色、带白色粗描边**的字体（增强复杂背景下的可读性）。
* **伤害跳字:** 出现时附带弹簧缩放动画 (Scale `0.5 -> 1.2 -> 1.0`)，然后匀速向上漂浮并改变透明度至淡出。
* **特殊状态:** * 若为暴击 (Critical)：字体再放大 1.5 倍，变为亮橙色。
  * 若为闪避/格挡：不显示伤害数字，直接向上飘出灰色文字“闪避!”或黄色文字“格挡!”。

---

## 4. 结算状态 (Post-Battle Modal)
动画播放完毕或玩家点击跳过后，屏幕中心变暗，弹层显示结算：
* **胜利 (Win):** 顶部展示胜利徽章/皇冠。按行清晰罗列：经验值、铜钱（带金币 Icon），如果有装备掉落，必须在正中央展示该装备的高清大图及品质框。
* **失败 (Defeat):** 整体色调变灰，仅提示“挑战失败，请提升实力”。
