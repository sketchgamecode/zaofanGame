# 项目计划书：大宋造反模拟器 (3天体验版 MVP)

## 1. 项目概述
* **目标:** 验证“换皮版 Shakes and Fidget (S&F)”核心放置点击玩法在国风/暗黑武侠题材下的玩家接受度。
* **开发模式:** 极简开发，仅实现前3天的游玩内容，后续系统使用占位符（Placeholder）。
* **技术栈:** Web H5 (React + TypeScript + Tailwind CSS)
* **数据存储:** 初期纯前端，使用 `localStorage` 或 `zustand` 缓存，无后端数据库。
* **部署:** 编译为静态文件，托管至 Cloudflare Pages (域名: sketchgame.net)。

## 2. Agent 分配与协作策略
* **Agent 1 (架构与数据):** 负责 `03_Data_Tables.md` 的落地，编写玩家实体、背包实体、任务实体的数据结构和本地存储逻辑。**严禁处理 UI。**
* **Agent 2 (前端与 UI):** 负责读取 Agent 1 的数据，结合 `02_Core_Gameplay.md` 的描述，使用 Tailwind CSS 绘制界面布局和占位图。
* **Agent 3 (玩法逻辑):** 负责任务倒计时、随机池抽取、战斗数值对比（极简版）等动态系统计算。

## 3. 里程碑与排期 (总计 2.5 天)

### Phase 1: 基础架构搭建 (0.5天)
- [ ] 初始化 React + TS 项目。
- [ ] 设定全局状态树（金币、经验、属性、装备）。
- [ ] 跑通数据的本地读写测试。

### Phase 2: 核心 UI 与角色系统 (1天)
- [ ] 搭建左右分栏的 PC/Mobile 自适应主框架。
- [ ] 实现【人物面板】：展示基础属性，跑通“消耗铜钱升级属性”的交互（数值指数递增）。
- [ ] 实现【行囊面板】：4个核心装备槽位，支持点击穿戴/替换。

### Phase 3: 核心玩法闭环 (0.5天)
- [ ] 实现【密谋据点】：随机生成 3 个任务（区分金币向/经验向/装备向）。
- [ ] 实现任务进度条与倒计时逻辑。
- [ ] 实现任务结算结算与掉落判定。

### Phase 4: 占位符与发版部署 (0.5天)
- [ ] 制作【演武场】、【攻打州府】、【聚义厅】的敬请期待界面。
- [ ] 接入占位美术资源（人物头像、装备 Icon）。
- [ ] 打包构建，推送到 Cloudflare 上线测试。
